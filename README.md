# Nash Agent

Nash Agent — инструмент для менеджеров продукта, который анализирует фичу или стратегию с точки зрения теории игр и ищет равновесие Нэша в многопользовательской игровой модели. Из кейса агент выделяет участников, возможные стратегии поведения участников, оценивает стратегии через LLM и собирает результат в управленческий отчёт.

Сервис помогает до начала разработки понять, какие акторы могут поддержать или сломать запуск отдельной фичи или продуктовой стратегии вцелом, где возникают риски, какие ограничения стоит поставить и какой следующий ход выглядит рациональным для менеджера продукта.

## Что умеет сервис

- Выделяет игроков, границы игры, стратегии и допущения из текстового описания кейса.
- Строит до 64 стратегических профилей и оценивает их.
- Рассчитывает равновесия Нэша, риски и стратегические ходы.
- Формирует развёрнутую статью с объяснением игры по предложенному кейсу.
- Собирает `План действий для продакт-менеджера`: что делать дальше и почему.
- Хранит историю проанализированных кейсов.
- Экспортирует полный анализ кейса в PDF-документ для печати.
- Позволяет выбрать провайдера LLM: локальная LLM через LM Studio или облачная через Yandex AI Studio.

## Архитектура

Проект — full-stack TypeScript-приложение.

- `client/`
  - React + Wouter.
  - Tailwind CSS и shadcn/ui.
  - TanStack Query для API-запросов.
  - Recharts для графиков.
  - Основные страницы:
    - `NewAnalysis.tsx` — создание кейса и live progress во время анализа.
    - `AnalysisView.tsx` — полный результат анализа, визуализации, статья агента, PM Decision Pack и ссылка на PDF.
    - `History.tsx` — список кейсов, статусы, длительность, количество профилей и фрагментов.
    - `Settings.tsx` — выбор LLM-провайдера и параметры подключения.

- `server/`
  - Express API.
  - SSE-стриминг ответа от LLM.
  - Очередь запросов к LLM, чтобы этапы выполнялись последовательно и управляемо.
  - Интеграция с OpenAI-compatible API для LM Studio и Yandex AI Studio.
  - Генерация PDF через `pdfkit`.

- `shared/`
  - Общая Drizzle-схема и TypeScript-типы для клиента и сервера.

- `SQLite + Drizzle`
  - Локальное хранение кейсов и результатов в `nash.db`.

## Пользовательский сценарий

1. Пользователь вводит название и описание кейса, а также контекст.
2. Агент готовит запрос к LLM и выделяет игроков, границы игры, стратегии и допущения.
5. Бэкенд моделирует кейс.
6. LLM генерирует развёрнутую статью с результатами на русском языке.
7. LLM собирает `План действий для продакт-менеджера`.
8. Результат сохраняется в истории и становится доступен как страница кейса и PDF-отчёт.

## LLM-провайдеры

Провайдер выбирается на странице `Настройки`.

### Локальная LLM через LM Studio

Подходит для приватной работы с чувствительными продуктовыми кейсами.

### Yandex AI Studio

Подходит для облачного режима, когда нужна более стабильная или производительная модель. API-ключ пользователь вводит на странице `Настройки`.

## Локальный запуск

Установите зависимости:

```bash
npm install
```

Создайте `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Запустите приложение:

```bash
npm run dev
```

По умолчанию сервис поднимается на:

```text
http://127.0.0.1:5000/#/
```

Порт можно изменить через `PORT`. В локальной разработке удобно использовать `PORT=5001`, если `5000` уже занят.

## Проверка

Проверка TypeScript:

npm run check
